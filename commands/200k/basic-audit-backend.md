> 백엔드 감사 → 이슈 생성 | context_tier: 200k

백엔드 코드를 감사하여 개선/수정이 필요한 항목을 발견하고, 이슈로 등록합니다.

인자: `$ARGUMENTS` (선택, 집중 영역 지정 예: `security`, `error-handling`. 없으면 전체 관심사 실행)

## Step 1: 기존 이슈 수집 + 라벨 사전체크

```bash
gh issue list --state open --limit 100 --json number,title,labels
"/c/Program Files/GitHub CLI/gh.exe" label list --json name
```

1. 이슈 목록을 이후 모든 발견과 대조한다. 이미 이슈가 있는 문제는 건너뛴다.
2. Step 4에서 사용할 라벨(`security`, `bug`, `enhancement` 등)이 존재하는지 확인한다. 없는 라벨은 `gh label create`로 미리 생성한다.

## Step 2: 관심사별 Explore Agent 병렬 실행

4개 관심사 × 관심사당 1개 Explore Agent(model: **sonnet**)를 **병렬로** 실행한다. `$ARGUMENTS`로 영역이 지정된 경우 해당 agent만 실행한다 (예: `security` → Agent 1만, `test` → Agent 3만, `hygiene` → Agent 4만).

**agent 수 조정:** 대상 파일이 많아 단일 agent의 컨텍스트로 부족할 경우, 해당 관심사를 디렉토리별로 분할하여 복수 agent를 실행한다.

### Agent 1: 보안 + 입력 검증

프로젝트에서 API 라우트, 인증 미들웨어, 입력 검증 관련 코드를 스스로 탐색하여 보안 취약점과 입력 검증 누락을 찾는다. 코드를 직접 읽고 문제를 특정하라.

### Agent 2: 에러 핸들링 + API 일관성

프로젝트에서 API 라우트, 서버사이드 로직, 데이터 레이어 코드를 스스로 탐색하여 에러 처리 누락, 응답 형식 불일치, 장애 대응 부재를 찾는다. 코드를 직접 읽고 문제를 특정하라.

### Agent 3: 테스트 커버리지 + 데이터 무결성

프로젝트에서 테스트 파일, 데이터 레이어, DB 스키마를 스스로 탐색하여 테스트 누락, 타입-스키마 불일치, 데이터 처리 오류를 찾는다. 코드를 직접 읽고 문제를 특정하라.

### Agent 4: 코드 위생

프로젝트의 소스 코드 전반을 탐색하여 코드 위생 문제를 찾는다. 코드를 직접 읽고 문제를 특정하라. 대상: 미사용 import/변수, dead code, console.log/debugger, TODO/FIXME, 네이밍 불일치.

### Agent 공통 지시

각 Agent에게 전달할 지시:
```
발견 항목을 아래 형식으로 반환하라:

### 발견 1: [문제 요약]
- 위치: `파일:라인`
- 근거: 확인됨 / 추정
- 코드:
  ```
  // 해당 라인 주변 3-5줄 스니펫
  ```
- 설명: 구체적으로 무엇이 왜 문제인지

### 발견 2: ...

규칙:
- 근거 "확인됨": 코드를 직접 읽고 문제를 특정한 경우 (파일:라인 필수)
- 근거 "추정": 코드 패턴상 문제가 있을 것 같지만 완전히 확인하지 못한 경우
- 추정이라도 의심되면 보고한다 (본체가 2차 판단)
- 이미 해결된 것은 보고하지 않는다
```

## Step 3: 발견 통합 + 이슈 그룹핑

Agent 결과를 수집하여:

1. **중복 제거** — 같은 파일:라인이 여러 agent에서 보고된 경우 병합
2. **기존 이슈 대조** — Step 1에서 수집한 open 이슈와 비교, 이미 있으면 스킵
3. **그룹핑 판단**:
   - 같은 패턴의 반복 → 하나의 이슈로 그룹
   - 고유한 문제 → 단독 이슈
4. **자동 필터링 — 이슈 생성 대상 결정**:
   - 근거 "확인됨" + 기존 이슈와 중복 아님 → **자동 생성**
   - 근거 "추정" → 본체가 해당 파일:라인을 직접 Read로 읽고 2차 판단 → 진짜면 생성, 아니면 버림
   - 기존 이슈와 겹침 → **생성하지 않음** (스킵 로그에 기록)

## Step 4: 이슈 생성

필터링을 통과한 항목에 대해 **Bash 병렬 호출**로 이슈를 일괄 생성한다.
에이전트를 띄우지 않고 본체가 직접 `gh issue create`를 병렬 Bash tool call로 실행한다.

```bash
gh issue create \
  --title "제목" \
  --body "$(cat <<'EOF'
## 문제

[감사에서 발견한 내용]

## 해당 위치

- `파일:라인` — 구체적 문제
- ...

## 제안

[수정 방향]

---
🤖 `/basic-audit-backend`에서 자동 생성됨
EOF
)" \
  --label "라벨"
```

라벨 매핑:
- 보안 관련 → `security`
- 에러 핸들링 → `bug` 또는 `refactor`
- 테스트 부재 → `test`
- API 일관성 → `refactor`

**실행 방식:** 이슈별로 독립된 Bash tool call을 만들어 한 메시지에서 병렬 실행. 에이전트 오버헤드 없이 빠르게 생성.

## Step 5: 결과 요약

```
## 감사 완료

- 탐색 영역: backend
- 발견 항목: N개
- 생성 이슈: M개
- 스킵 (기존 이슈): K개
- 추정 → 버림: L개

| 이슈 | 제목 | 라벨 |
|------|------|------|
| #301 | ... | security |
| #302 | ... | refactor |

다음 단계: `/spec 301 302 303` 으로 스펙 생성
```
