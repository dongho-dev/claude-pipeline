> context_tier: 1m

명세가 있는 open 이슈를 분석하여 실행 계획을 수립합니다. 충돌 분석, MIS 계산, 배치 구성, 리뷰 레벨 지정까지 수행합니다.

## Step 1: 이슈 + PR 수집

이미 컨텍스트에 이슈 목록이 있으면 재수집을 생략한다. 없으면:

```bash
gh issue list --state open --limit 200 --json number,title,labels,body
gh pr list --state open --json number,title,headRefName
```

이미 PR이 올라간 이슈는 "PR 진행 중"으로 표시하고 MIS 계산에서 제외한다.

## Step 2: Write 파일 매핑 (2트랙)

**워커 실행 가능** 이슈만 대상으로, 명세 유무에 따라 처리 방식이 다르다.

### 명세 판별 기준
이슈 본문에 `### 변경 파일 목록` 또는 `### 변경 사항` 또는 `**Write 파일:**` 섹션이 존재하면 **명세 있음**.

### Track A: 명세 있는 이슈 (파싱)
- 이슈 본문의 변경 파일 섹션에서 파일 경로를 정규식으로 추출
- AI 코드 탐색 불필요 → 이슈 수에 무관하게 O(1) 컨텍스트
- 패턴: 백틱 안의 경로 또는 `- ` 뒤의 경로 (`src/...`, `lib/...`, `app/...` 등)

### Track B: 명세 없는 이슈 (제외)
- **"명세 없음"으로 표시하고 MIS 계산에서 제외**
- `/basic-spec`으로 명세 먼저 작성 권고

## Step 3: 충돌 분석 + MIS 계산

**명세 있는 이슈만** 대상으로 MIS를 계산한다.

1. Write 파일이 겹치는 이슈 쌍을 모두 찾는다 (충돌 간선)
2. 충돌 그룹에서 **priority 라벨**로 우선순위를 비교하여 이슈를 선택한다 (high > medium > low). 동일 priority면 최소 차수 노드를 선택한다.
3. 충돌 없는 이슈는 모두 포함한다
4. **최대 독립 집합 (Maximum Independent Set)** 을 구성한다

MIS 계산은 node/python 스크립트로 수행 (Greedy: priority 높은 노드 우선 → 동점 시 최소 차수 노드 선택 → 이웃 제거 → 반복).

## Step 4: 실행 방식 결정

| 상황 | 방식 |
|------|------|
| 이슈 1~3개, 순차 가능 | 직접 처리 |
| 이슈 4개 이상 + 파일 겹침 없음 | `/basic-agents` 권장 |
| 이슈들 간 강한 의존성 | 직접 처리 (순차) |
| 복잡한 아키텍처 결정 포함 | 직접 처리 + 사용자 상의 |

## Step 5: 배치 구성

MIS 결과를 바탕으로 배치를 구성한다:
1. **배치당 상한 5~6개**
2. 명세에 이슈 간 의존이 명시된 경우, 같은 배치에 순서대로 배치

## Step 6: 리뷰 레벨 지정

각 이슈에 L1/L2 리뷰 레벨을 배정한다:

| 레벨 | 대상 |
|------|------|
| **L1 (구현 검증)** | 단일 파일 수정, API endpoint, UI 컴포넌트, 버그 수정 |
| **L2 (아키텍처 검증)** | 다중 파일 연쇄 변경, 데이터 흐름 변경, 새 패턴 도입, 보안 |

**L2 자동 승격**: Write 파일 3개 이상, 프로젝트 핵심 파일(타입 정의, 데이터 레이어, 미들웨어, DB 스키마) 변경, 또는 `priority:high` 라벨.

## Step 7: 보고

```
## 실행 계획

**추천 방식: 직접 처리 / basic-agents**
→ 이유: ...

### 배치 구성

Batch A (fix/batch-a):
  - Issue #N: 설명 → 변경 파일들 [L1] [priority:medium]
  - Issue #M: 설명 → 변경 파일들 [L2] [priority:high]
    근거: priority:high → L2 자동 승격

Batch B (fix/batch-b):
  - Issue #K: 설명 → 변경 파일들 [L1] [priority:low]

Write 파일 겹침: 없음 ✓

### 제외된 이슈

| # | 제목 | 제외 이유 | 비고 |
|---|------|----------|------|
| ... | ... | 충돌 (#XX와 겹침, priority 낮아 제외) | |
| ... | ... | 명세 없음 | /basic-spec 필요 |

이 계획으로 `/basic-agents` 실행할까요?
```
